亚马逊运营

基础介绍

亚马逊主流运营模式（按发货方式区分）

  • FBA（Fulfillment by Amazon）模式：批量配货并发货至亚马逊官方仓库，出单后由亚马逊进行发货；
  • FBM（Fulfillment by Merchant）模式：根据库存所在地分为【国内直发】和【第三方海外仓发货】两种：
    1. 国内直发模式：无需囤货，批量上传大量产品链接，出单后到供应商处采购，再发往国外；
    2. 第三方海外仓发货模式：
      • 囤货至第三方海外仓，出单后由第三方海外仓发货；常见FBA库容限制和易融产品(每年5月1号-10月中旬亚马逊仓库不接收)
      • 和供销平台合作，销售供销平台的产品，作为中间商挣差价，不需要囤货；
        如赛赢分销、大件云仓等；

精品、精铺、泛铺（按经营方式区分）

  • 精品：自行开发一款产品（私模）市面上没有同款，完美规避同质化竞争；
    以FBA模式为主，资金投入风险较高
  • 精铺：细致分析市场数据选品，只做现货。做好每一个运营细节
    FBA、FBM两种模式都行，FBA收益最大，但投入较高
  • 泛铺：利用ERP（Enterprise Resource Planning）软件或者纯手工上传大量产品，没有选品没有运营。
    全靠数量取胜；
    有手就行，对于小卖家只适合FBM模式，侵权挂账号风险较大

Avg. Customer Review排序规则

  • 评分高低
    默认情况下，商品按平均评分从高到低排序。例如：4.8星的商品会排在4.5星商品之前。

  • 评价数量

    如果两个商品评分相同，评价数量更多的商品会优先展示例如：4.5星(1000条评价)的商品会排在4.5 星(100条评价)的商品之前。

  • 其他因素

    • 近期评价：较新的评价可能对排序有更大影响。

    • Verified Purchase（已验证购买）：来自真实购买者的评价可能权重更高

    • 评价质量：包含文字、图片或视频的详细评价可能对排序有一定影响。

NewestArrivals排序规则
单纯按上架时间以及listing的更新时间排序，用于了解某个细分品类下，最近上架的一些新品

BestSellers排序规则

专业术语

变体

变体Variations:又称父/子关系
亚马逊平台上的商品管理概念，指同一款产品因颜色、尺寸、等属性差异而形成的多版本商品集合，通过父体
材质(Parent)和子体(Child)关系整合在一个商品详情页(Listing)

  • 父体Parent：用于关联子商品，虚拟概念且前台不展示
  • 子体Child：每个父体下关联的商品
  • 变体主题：父商品与子商品之间的关系（如鞋子的尺码、手机的型号）(颜色、尺码、材质.)

ASIN

Amazon Standard Identification Number，亚马逊标准识别号，产品发布之后，亚马逊会随机生成一串10位由数字和字母所组成的编码通常以BO为开头。这串编码具有唯一性，就好比产品的身份证。
俗称：B0码
作用：快速定位产品(设置优惠促销、广告、反查竞品.)

SKU

Stock Keeping Unit 库存单位，便于管理识别商品与定位商品
SKU与ASIN的区别
ASIN:
相当于某人的身份证号码，具有唯一性，由系统生成
SKU：相当于某人的名字或者外号，不具备唯一性，卖家可以自定义SKU编码

FNSKU

使用FBA服务时生成的一种独特商品编码，便于亚马逊FBA仓储系统标识、管理和追踪库存
当产品以FBA模式进行配送时，由亚马逊FBA系统生成，通常以”XO”开头，也称”XO码”

自发货没有这个码

跟卖

指卖家在亚马逊平台上同一条listing上销售与你完全相同的商品，与你共享同一个产品页面，从而争夺购物车(Buy Box)
和订单的行为。这是亚马逊平台允许的机制，容易引发价格战和恶性竞争。

比如：A卖家创建了一条listing，B卖家也想卖这款产品，B卖家为了节省创建listing的时间，直接在A卖家创建的listing中进行跟卖；

BuyBox黄金购物车

亚马逊Listing的一个核心功能，位于页面右侧，买家可以直接点击“加入购物车或’立即购买’完成下单。
谁赢得BuyBoX，谁就能获得80%以上的订单！据统计，约82%的亚马逊交易通过BuyBox完成）

没有购物车的影响：

  1. 买家购买流程复杂：需多次点击页面才能完成下单，导致转化下降
  2. 产品曝光减少：
    1.系统不会推荐无购物车的listing
    2.无购物车无法开启站内广告，以及无法使用站内促销工具
  3. 流量减少、单量减少：曝光减少、转化下降的连锁反应，导致排名下降
  4. Listing控制权丢失：
    在被跟卖的情况下，被抢夺购物车，也会导致listing失去控制权

如何获得购物车

  • 使用FBA：FBA模式下Listing基本都会有购物车
  • 使用专业销售计划：每月39.99美金服务费
  • 优化卖家绩效：减少订单取消、延迟发货和差评。(FBM模式)
  • 保持库存充足：避免因断货导致BuyBox丢失。
  • 完成品牌备案：防止跟卖，被人抢夺购物车。

Listing变狗的常见原因

侵权投诉（最常见）

  • 品牌侵权：被商标持有者投诉
  • 专利/版权侵权：如图片、文案抄袭
  • 假货投诉：被买家或品牌方举报卖假货

违反亚马逊政策

  • 滥用变体：违规合并变体

账户问题

  • 账号被封：导致所有Listing下架
  • 绩效不达标：ODR过高（订单缺陷率）、迟发率超标等（主要针对FBM）

如何避免变狗

  • 确保商品合规：不卖侵权、违禁品
  • 使用原创图片和文案：避免抄袭同款listing信息
  • 提前申请类目审核：如FDA、CPC、FCC认证
  • 避免违规操作：不刷单、不滥用变体，不乱玩“黑科技”
  • 监控账户健康：及时处理投诉和绩效问题

什么是GTIN

Global Trade Item Number，全球贸易项目代码。是全球通用的商品编码标准，用于唯一标识产品。在亚马逊上，GTIN通常指UPC、EAN、IISBN或JAN等条形码，是创建商品Listing时必须提供的编码之一

编码类型

UPC：美国统一代码委员会制定的一种商品用条码，主要用于美国和加拿大地区
EAN:国际物品编码协会制定的一种商品用条码，通用于全世界，主要用于欧洲
JAN：日本工业标准(JIS)制定，主要用于亚马逊日本站
ISBN：国际标准书号，专门为识别图书等文献而设计的国际编号
GCID：全球目录编码，完成品牌备案后，系统进行分配

FDA认证

FDA 是食品药品监督管理局(Food and Drug Administration)的简称，美国政府在健康与人类服务部(DHHS)和公共卫生部(PHS)中设立的执行机构之一。机构职责是确保美国本国生产或进口的食品、化妆品、药物、生物制剂、医疗设备和放射产品的安全。卖家售卖的产品如果涉及到食品、药品、或者任何需要与人体接触的产品，均被要求提交此类认证，否则会面临产品强制下架的风险。
商品示例：食品、餐具、化妆品、药物、生物制剂、医疗设备和放射产品等。

CPC认证

美国消费品安全法案(CPSA)规定，儿童产品的制造商和进口商必须基于CPSC认可的实验室的测试结果，提供书面的儿童产品证书(Children’s Product Certificate,CPC)，以证明其产品满足相应的儿童产品安全条例。
CPC适用于所有以12岁及以下儿童为主要目标使用对象的产品，如玩具、摇篮、儿童服装等，如在美国本地生产则由制造商负责提供，如在其他国家生产则由进口商负责提供

商品示例：电动玩具、摇铃玩具、奶嘴、儿童服装、童车、童床、围栏、背带、安全座椅、自行车头盔等。

FCC认证

FCC是美国联邦通信委员会(Federal Communications Commission)的简称。FCC 通过控制无线电广播、电视、电信、卫星和电缆来协调国内和国际的通信。许多无线电应用产品通讯产品和数字产品要进入美国市场，都要求FCC的认可。
商品示例：个人电脑及其周边设备、家用电器设备、电动工具、音视频产品、灯具、无线产品、玩具类产品、安防产品、工业机械等。

Listing编写和优化

标题

产品标题的重要性

标题即亚马逊产品的商品名称
使用准确、简洁的“关键词”，介绍产品。
同时通过标题中所包含的”关键词”触发搜索引擎抓取产品，展示给搜索关键词”的消费者！

关键词即买家所搜索的内容
买家通过什么词搜索要购买的产品，那么这个词就是关键词。“有人搜索的词才是关键词“

影响

影响曝光、流量、转化：对产品销售有直接影响：
产品标题是消费者在亚马逊平台上搜索产品时首先看到的信息。
如果标题中的关键词不符合用于的搜索习惯，产品很难被消费者找到，从而导致曝光和流量下降。

产品关键词类型

  • 大词/流量词/泛词：与产品的相关性比较低，同时搜索量大（连衣裙/dress)

  • 精准词：与产品相关性比较高，同时搜索量较小（红色连衣裙/reddress)

  • 精准长尾词：与产品相关性极高，同时搜索量相对较小(红色长款连衣裙/red long dress)

  • 核心关键词：直接描述产品主要属性的关键词，搜索量大，相关性高

不同关键词类型数据差别原因

  • 用户搜索习惯不同：搜索量
    购买此类产品的用户，习惯搜索精准词、精准长尾词，导致比大词的搜索量高（如手机壳）

  • 卖家主推关键词不同：PPC竞价
    在相对比较成熟且竞争较大的市场中，卖家运营能力都很强。通过关键词数据了解用户的搜索习惯，所以多数卖家都不投放大词，转而投放精准长尾词，导致精准长尾词的PPC竞价较高

1-介绍

理解

1．抽象表达:

  • Promise是一门新的技术（ES6规范）

  • 从功能上来说：promise对象用来封装一个异步操作并可以获取其成功/失败的结果值

优势

2-基本使用

Promise的状态PromiseState

一个 Promise 必然处于以下几种状态之一：

  • 待定（pending）: 初始状态，既没有被兑现，也没有被拒绝。
  • 成功（fulfilled）: 意味着操作成功完成。
  • 失败（rejected）: 意味着操作失败。

一个 Promise 的状态只能由 pending 变为 fulfilled，或者变为 rejected，且只能改变一次。

链式调用：Promise 对象具有 then 和 catch 方法来处理下一步的操作。因为 Promise.prototype.then 和 Promise.prototype.catch 方法返回的是 Promise，所以它们可以被链式调用。

3-详细使用

Promise API

Promise 构造函数

Promise 构造函数接收一个执行器 executor，来启动这个 Promise。

参数 :

resolve 和 reject，分别表示操作成功和操作失败。

  • executor 函数：执行器 (resolve, reject) => { }
  • resolve 函数：内部定义成功时调用的函数 value => {}
  • reject 函数：内部定义失败时调用的函数 reason => {}

返回值：

当通过 new 关键字调用 Promise 构造函数时，它会返回一个 Promise 对象。当 resolveFunc 或者 rejectFunc 被调用时，该 Promise 对象就会变为已解决（resolved）。

注意：executor会在Promise内部立即同步调用.异步操作在执行器中执行

  • new Promise(executor) 这行代码执行时，**executor 会立刻运行**（不是等到后面 .then() 被调用才运行）。
  • executor 是同步执行的，但它的内部可以启动异步操作（比如 setTimeout、fetch、fs.readFile 等）。
  • 异步操作的结果通过调用 resolve(value) 或 reject(reason) 来通知 Promise 状态变更。

执行结果：

  • 同步部分：Promise 构造函数的调用、executor 的初始执行、.then() 方法的调用。
  • 异步部分：
    • executor 中启动的异步操作（如网络请求、定时器等）；
    • .then() / .catch() 中回调函数的实际执行（总是异步，走微任务队列）。

Promise.prototype.then

Promise.prototype.then 方该方法返回的结果依然是一个 Promise 对象，意味着我们可以进行链式调用。

语法：

参数：

  • onFulfilled 函数：成功的回调函数 (value) => {}
  • onRejected 函数：失败的回调函数 (reason) => {}

返回值：
如果 then 中的回调函数：

  • 返回了一个值，那么 then 返回的 Promise 将会成为接受状态（fulfilled），并且将返回的值作为接受状态的回调函数的参数值。
  • 没有返回任何值，那么 then 返回的 Promise 将会成为接受状态（fulfilled），并且该接受状态的回调函数的参数值为 undefined。
  • 抛出一个错误，那么 then 返回的 Promise 将会成为拒绝状态（rejected），并且将抛出的错误作为拒绝状态的回调函数的参数值。
  • 返回一个已经是接受状态（fulfilled）的 Promise，那么 then 返回的 Promise 也会成为接受状态，并且将那个 Promise 的接受状态的回调函数的参数值作为该被返回的Promise的接受状态回调函数的参数值。

  • 返回一个未定状态（pending）的 Promise，那么 then 返回 Promise 的状态也是未定的，并且它的终态与那个 Promise 的终态相同；同时，它变为终态时调用的回调函数参数与那个 Promise 变为终态时的回调函数的参数是相同的。

Promise的几个关键问题

1-如何修改对象的状态

  • resolve(value)：如果当前是 pending 就会变为 resolved
  • reject(reason)：如果当前是 pending 就会变为 rejected
  • 抛出异常：如果当前是 pending 就会变为 rejected

2-一个promise指定多个成功/失败回调函数，都会调用吗？

当 promise 改变为对应状态时都会调用。

3-改变promise状态和指定回调函数谁先谁后？

  1. 都有可能，正常情况下是先指定回调再改变状态，但也可以先改状态再指定回调
  2. 如何先改状态再指定回调?
    • 在执行器中直接调用 resolve()/reject()
    • 延迟更长时间才调用 then()
  3. 什么时候才能得到数据?
    • 如果先指定的回调，那当状态发生改变时，回调函数就会调用，得到数据
    • 如果先改变的状态，那当指定回调时，回调函数就会调用，得到数据

4-promise.then() 返回的新promise 的结果状态由什么决定?

  • 简单表达：由 then() 指定的回调函数执行的结果决定
  • 详细表达：
    1. 如果抛出异常，新 promise 变为 rejected, reason为抛出的异常
    2. 如果返回的是非 promise 的任意值，新promise变为 resolved，value 为返回的值
    3. 如果返回的是另一个新 promise，此 promise 的结果就会成为新 promise 的结果

5-promise 如何串连多个操作任务?

  • promise的 then() 返回一个新的 promise，可以看成 then() 的链式调用
  • 通过 then 的 链式调用串连多个同步/异步任务

.then() 中可以处理同步任务，也可以处理异步任务。处理异步任务时需要新创建一个 Promise。

6-promise异常传透？

在 Promise 的链式调用链中，当所有的 .then 中都没有指定错误处理的回调，则前面出现的异常会在最后失败的回调中处理。

注意：

  • .catch 所谓的异常穿透并不是一次失败状态就触发 catch，而是一层一层的传递下来的
  • 异常穿透的前提条件是所有的 .then 都没有指定失败状态的回调函数。
  • 如果 .catch 前的所有 .then 都指定了失败状态的回调函数，.catch 就失去了意义。

await表达式

  • await 右侧的表达式一般为 promise 对象,但也可以是其它的值
  • 如果表达式是 promise 对象，await 返回的是 promise 成功的值
